# 卡片管理指南

## 📋 概述

本指南介绍如何通过Web管理界面管理NFC卡片，包括添加、编辑、删除和查询卡片。

## 🎯 功能特性

### ✅ 已实现功能

- **添加卡片** - 注册新的NFC卡片
- **编辑卡片** - 修改卡片信息和权限
- **删除卡片** - 移除不再使用的卡片
- **查询卡片** - 搜索和筛选卡片列表
- **时间段控制** - 设置卡片可用的时间段
- **设备权限** - 限制卡片可访问的设备
- **访问时间范围** - 设置卡片的有效期
- **缓存控制** - 控制卡片是否可被设备缓存

## 🚀 快速开始

### 1. 访问卡片管理页面

1. 登录Web管理界面
2. 点击左侧导航栏的"卡片管理"
3. 进入卡片管理页面

### 2. 添加卡片

#### 方法1: 手动输入UID

1. 点击"添加卡片"按钮
2. 填写卡片信息：
   - **卡片UID**: 8-14位十六进制字符（必填）
   - **姓名**: 持卡人姓名（必填）
   - **其他选项**: 根据需要配置
3. 点击"添加卡片"提交

#### 方法2: 从ESP32读取UID

1. 将NFC卡片靠近ESP32设备
2. 查看ESP32串口输出或访问日志
3. 复制显示的UID
4. 在Web界面添加卡片时粘贴UID

### 3. 编辑卡片

1. 在卡片列表中找到要编辑的卡片
2. 点击该卡片行的"编辑"按钮（铅笔图标）
3. 修改卡片信息
4. 点击"保存更改"

### 4. 删除卡片

1. 在卡片列表中找到要删除的卡片
2. 点击该卡片行的"删除"按钮（垃圾桶图标）
3. 确认删除操作

## 📝 字段说明

### 基本信息

#### 卡片UID（必填）
- **格式**: 8-14位十六进制字符
- **示例**: `04A1B2C3D4E5F6`
- **说明**: NFC卡片的唯一标识符
- **注意**: 
  - 添加后不可修改
  - 必须唯一，不能重复
  - 自动转换为大写

#### 姓名（必填）
- **格式**: 1-100个字符
- **示例**: `张三`
- **说明**: 持卡人姓名或卡片描述

### 权限控制

#### 启用卡片
- **类型**: 开关
- **默认**: 启用
- **说明**: 
  - 启用：卡片可以正常使用
  - 禁用：卡片无法通过验证

#### 允许缓存
- **类型**: 开关
- **默认**: 允许
- **说明**:
  - 允许：设备可以缓存此卡片信息（离线可用）
  - 不允许：设备必须每次都连接云端验证

### 时间控制

#### 访问开始时间
- **格式**: 日期时间（YYYY-MM-DD HH:MM）
- **示例**: `2024-01-01 00:00`
- **说明**: 卡片开始生效的时间
- **留空**: 表示不限制开始时间

#### 访问结束时间
- **格式**: 日期时间（YYYY-MM-DD HH:MM）
- **示例**: `2024-12-31 23:59`
- **说明**: 卡片失效的时间
- **留空**: 表示不限制结束时间

#### 时间段限制
- **格式**: `HH:MM-HH:MM`，多个时间段用逗号分隔
- **示例**: `09:00-12:00, 14:00-18:00`
- **说明**: 每天允许访问的时间段
- **留空**: 表示全天24小时可用
- **规则**:
  - 时间格式必须为24小时制
  - 开始时间必须早于结束时间
  - 多个时间段用英文逗号分隔

### 设备控制

#### 允许的设备
- **格式**: 设备ID列表，用逗号分隔
- **示例**: `door-001, door-002`
- **说明**: 限制卡片只能在指定设备上使用
- **留空**: 表示可以在所有设备上使用
- **注意**: 设备ID必须已在系统中注册

## 💡 使用场景

### 场景1: 普通员工卡

```
卡片UID: 04A1B2C3D4E5F6
姓名: 张三
启用: ✅
允许缓存: ✅
访问开始时间: （留空）
访问结束时间: （留空）
时间段限制: 09:00-18:00
允许的设备: （留空 - 所有设备）
```

**说明**: 
- 全年有效
- 工作时间（9:00-18:00）可用
- 可以访问所有门禁设备
- 允许设备缓存（离线可用）

### 场景2: 临时访客卡

```
卡片UID: 04B2C3D4E5F6A7
姓名: 访客-李四
启用: ✅
允许缓存: ❌

访问结束时间: 2024-05-03 18:00
时间段限制: （留空）
允许的设备: door-001
```

**说明**:
- 仅当天有效
- 全天可用
- 只能访问前门（door-001）
- 不允许缓存（必须实时验证）

### 场景3: 夜班员工卡

**说明**:
- 全年有效
- 夜间时段（22:00-次日6:00）可用
- 可以访问所有设备
- 允许缓存

### 场景4: 管理员卡

```

启用: ✅

允许的设备: （留空）
```

- 全年有效

- 可以访问所有设备
- 允许缓存

## 🔍 查询和筛选

### 搜索卡片

在搜索框中输入关键词，可以搜索：
- 卡片UID（部分匹配）
- 持卡人姓名（部分匹配）

### 筛选状态

使用状态筛选器：
- **全部状态**: 显示所有卡片
- **已启用**: 只显示启用的卡片
- **已禁用**: 只显示禁用的卡片

### 分页浏览

- 每页显示20条记录
- 使用底部分页控件切换页面
- 显示总记录数

## ⚠️ 注意事项

### 卡片UID

1. **格式要求**
   - 必须是8-14位十六进制字符
   - 只能包含0-9和A-F
   - 自动转换为大写

2. **唯一性**
   - 每个UID只能注册一次
   - 重复UID会提示错误

3. **不可修改**
   - UID一旦添加不能修改
   - 如需更改，只能删除后重新添加

### 时间段格式

1. **正确格式**
   ```
   ✅ 09:00-12:00
   ✅ 09:00-12:00, 14:00-18:00
   ✅ 00:00-23:59
   ```

2. **错误格式**
   ```
   ❌ 9:00-12:00        (小时必须两位)
   ❌ 09:00-12:00;14:00-18:00  (用逗号不是分号)
   ❌ 12:00-09:00       (开始时间晚于结束时间)
   ```

### 设备ID

1. **必须存在**
   - 设备ID必须已在系统中注册
   - 不存在的设备ID会提示错误

2. **格式要求**
   ```
   ✅ door-001
   ✅ door-001, door-002
   ✅ door-001, gate-main
   ```

### 访问时间范围

1. **时间顺序**
   - 开始时间必须早于结束时间
   - 否则会提示错误

2. **时区**
   - 使用本地时区
   - 显示格式根据浏览器设置

## 🔧 故障排查

### 问题1: 添加卡片失败

**症状**: 提示"UID already exists"

**原因**: 该UID已被注册

**解决**:
1. 检查是否重复添加
2. 在卡片列表中搜索该UID
3. 如果是误删，可以重新添加

---

**症状**: 提示"Invalid UID format"

**原因**: UID格式不正确

**解决**:
1. 确认UID是8-14位
2. 确认只包含0-9和A-F
3. 移除空格和特殊字符

---

**症状**: 提示"Device ID does not exist"

**原因**: 指定的设备ID不存在

**解决**:
1. 检查设备ID拼写
2. 在设备管理页面确认设备已注册
3. 或者留空允许所有设备

### 问题2: 卡片无法使用

**症状**: 刷卡后提示"Access DENIED"

**可能原因**:
1. 卡片被禁用
2. 不在访问时间范围内
3. 不在允许的时间段内
4. 设备不在允许列表中

**解决**:
1. 检查卡片状态是否启用
2. 检查当前时间是否在有效期内
3. 检查当前时间是否在允许的时间段内
4. 检查当前设备是否在允许列表中

### 问题3: 离线时卡片不可用

**症状**: WiFi断开后卡片无法使用

**原因**: 卡片未被缓存

**解决**:
1. 检查卡片的"允许缓存"选项是否启用
2. 在线时至少刷一次卡，让设备缓存
3. 确认设备缓存未满（最多50张）

## 📊 批量操作

### 批量添加卡片

目前不支持批量添加，需要逐个添加。

**建议流程**:
1. 准备卡片清单（Excel/CSV）
2. 逐个读取UID并记录
3. 通过Web界面逐个添加
4. 验证每张卡片

### 批量修改卡片

目前不支持批量修改，需要逐个修改。

**建议流程**:
1. 使用筛选功能找到目标卡片
2. 逐个编辑修改
3. 验证修改结果

## 🔐 安全建议

1. **定期审查**
   - 定期检查卡片列表
   - 删除不再使用的卡片
   - 禁用临时卡片

2. **权限最小化**
   - 只授予必要的权限
   - 使用时间段限制
   - 使用设备限制

3. **访客管理**
   - 访客卡设置有效期
   - 访客卡不允许缓存
   - 访客卡限制设备

4. **日志监控**
   - 定期查看访问日志
   - 关注异常访问
   - 及时处理问题
