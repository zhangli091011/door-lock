# PN532 NFC模块故障排查指南

## 问题：I2C通信失败

### 错误信息
```
E (5852) i2c.master: I2C transaction unexpected nack detected
E (5853) i2c.master: I2C transaction failed
```

## 原因分析

这个错误表示ESP32无法与PN532模块通信，可能的原因：

1. **接线错误** - SDA/SCL接反或松动
2. **模式设置错误** - PN532未设置为I2C模式
3. **供电问题** - 5V供电不足或不稳定
4. **引脚冲突** - GPIO引脚被其他设备占用
5. **模块故障** - PN532模块损坏

## 解决方案

### 步骤1: 检查接线

#### 标准接线（ESP32-S3 + PN532）

```
ESP32-S3          PN532
--------          -----
GPIO 8    <--->   SDA
GPIO 9    <--->   SCL
5V        <--->   VCC
GND       <--->   GND
```

#### 检查清单

- [ ] SDA连接到GPIO 8
- [ ] SCL连接到GPIO 9
- [ ] VCC连接到5V（不是3.3V！）
- [ ] GND连接到GND
- [ ] 所有连接牢固，没有松动
- [ ] 杜邦线质量良好，没有断线

#### 常见错误

❌ **错误1**: SDA和SCL接反
```
ESP32-S3          PN532
--------          -----
GPIO 8    <-X->   SCL  (错误！)
GPIO 9    <-X->   SDA  (错误！)
```

❌ **错误2**: 使用3.3V供电
```
ESP32-S3          PN532
--------          -----
3.3V      <-X->   VCC  (错误！PN532需要5V)
```

✅ **正确**: 使用5V供电
```
ESP32-S3          PN532
--------          -----
5V        <--->   VCC  (正确！)
```

### 步骤2: 检查PN532模式设置

PN532模块有3种通信模式，必须设置为**I2C模式**。

#### 模式开关位置

PN532模块背面有两个拨码开关（SW1和SW2）：

```
┌─────────────────┐
│   PN532 Module  │
│                 │
│  SW1  SW2       │
│  [ ]  [ ]       │
│   0    1        │
└─────────────────┘
```

#### 模式设置表

| 模式 | SW1 | SW2 | 说明 |
|------|-----|-----|------|
| HSU (UART) | ON | OFF | 串口模式 |
| **I2C** | **OFF** | **ON** | **I2C模式（我们需要的）** |
| SPI | OFF | OFF | SPI模式 |

#### 设置步骤

1. 断开PN532电源
2. 找到模块背面的拨码开关
3. 设置为：
   - SW1: **OFF** (向下或向0)
   - Sw2: **ON** (向上或向1)
4. 重新上电

#### 验证模式

正确设置I2C模式后，PN532的I2C地址应该是 **0x24**。

### 步骤3: 运行I2C扫描程序

使用提供的I2C扫描程序检测设备：

1. **上传扫描程序**
   ```
   打开 esp32-firmware/i2c_scanner.ino
   上传到ESP32-S3
   ```

2. **查看串口输出**
   ```
   打开串口监视器（115200波特率）
   查看扫描结果
   ```

3. **预期结果**
   ```
   ✅ Found 1 I2C device(s)!
   
   Device addresses found:
     - 0x24 (PN532 - I2C mode)
   
   ✅ PN532 detected at 0x24 (correct I2C address)
   ```

4. **如果没有找到设备**
   ```
   ❌ No I2C devices found!
   
   Troubleshooting:
   1. Check wiring
   2. Check PN532 mode switches
   3. Check power supply
   ```

### 步骤4: 检查供电

PN532需要稳定的5V供电。

#### 供电要求

- **电压**: 5V ±0.5V
- **电流**: 最大150mA（读卡时）
- **来源**: ESP32的5V引脚或外部5V电源

#### 供电问题症状

- I2C通信失败
- 模块工作不稳定
- 读卡距离变短
- 模块发热

#### 解决方案

1. **使用USB供电**
   - 确保USB线质量良好
   - 使用电脑USB 3.0接口或充电器

2. **使用外部电源**
   ```
   外部5V电源 --> PN532 VCC
   外部GND    --> PN532 GND + ESP32 GND
   ```

3. **添加电容**
   - 在VCC和GND之间并联100μF电容
   - 减少电源波动

### 步骤5: 更换I2C引脚

如果GPIO 8/9有问题，可以尝试其他引脚。

#### 可用的I2C引脚（ESP32-S3）

推荐引脚对：
- GPIO 8 (SDA) + GPIO 9 (SCL) - 默认
- GPIO 1 (SDA) + GPIO 2 (SCL)
- GPIO 3 (SDA) + GPIO 4 (SCL)
- GPIO 5 (SDA) + GPIO 6 (SCL)

#### 修改配置

编辑 `config.h`:
```cpp
// 原配置
#define PN532_SDA_PIN 8
#define PN532_SCL_PIN 9

// 改为
#define PN532_SDA_PIN 1
#define PN532_SCL_PIN 2
```

重新编译上传。

### 步骤6: 测试PN532模块

#### 使用Arduino示例程序

1. 打开Arduino IDE
2. 文件 > 示例 > Adafruit PN532 > readMifare
3. 修改引脚配置
4. 上传并测试

#### 使用万用表测试

1. **测试供电**
   - 红表笔接VCC，黑表笔接GND
   - 应该显示约5V

2. **测试I2C信号**
   - 使用示波器或逻辑分析仪
   - 查看SDA和SCL是否有信号

### 步骤7: 检查模块是否损坏

#### 损坏迹象

- 模块发热严重
- 有烧焦味道
- 芯片表面有损伤
- 更换所有接线后仍无法工作

#### 测试方法

1. **视觉检查**
   - 检查芯片是否有裂纹
   - 检查焊点是否完好
   - 检查电路板是否有烧痕

2. **更换模块**
   - 使用另一个PN532模块测试
   - 如果新模块工作正常，说明原模块损坏

## 常见问题FAQ

### Q1: 为什么必须使用5V供电？

A: PN532芯片设计工作电压为5V，虽然某些模块声称支持3.3V，但实际使用中3.3V供电会导致：
- 通信不稳定
- 读卡距离变短
- 功能异常

### Q2: 可以使用其他I2C引脚吗？

A: 可以。ESP32-S3的大部分GPIO都支持I2C功能，但建议：
- 避免使用GPIO 0（BOOT按钮）
- 避免使用GPIO 19/20（USB）
- 避免使用GPIO 43/44（UART）

### Q3: I2C扫描找到0x48而不是0x24？

A: 这说明PN532处于HSU（UART）模式，不是I2C模式。
- 检查模式开关：SW1=OFF, SW2=ON
- 断电后重新设置
- 重新上电测试

### Q4: 扫描找到多个设备怎么办？

A: 这是正常的，I2C总线可以连接多个设备。只要0x24存在就说明PN532正常。

### Q5: 模块工作一段时间后失效？

A: 可能是供电不足或过热：
- 检查USB线和电源
- 添加散热片
- 减少读卡频率
